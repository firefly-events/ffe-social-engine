
# Swarm Director Dashboard - Feature Walkthrough and API Documentation

This document provides an overview of the Swarm Director Dashboard, its features, and details the API endpoints it interacts with.

## 1. Feature Walkthrough

The Swarm Director Dashboard is a React-based Single Page Application (SPA) built with Next.js, designed for real-time monitoring and control of the swarm director orchestration.

### 1.1. Live Status

- **Purpose:** Displays the real-time connection status to the Director's WebSocket endpoint and the latest messages received from it. It also provides manual controls to interact with the Director.
- **Key Elements:**
    - WebSocket connection status (Connected/Disconnected).
    - Latest JSON message received from the Director's `/ws` endpoint.
    - **Manual Controls:** Buttons to trigger a new cycle, pause/resume orchestration, cancel an agent, or force garbage collection (future features).
- **Interactions:** Users can view live updates and send commands to the Director.

### 1.2. Cycle History

- **Purpose:** Visualizes the history of execution cycles, including start/end times and key events.
- **Key Elements:**
    - A timeline chart (placeholder) to represent cycle events.
    - Dummy data is currently used for demonstration.
- **Interactions:** Future enhancements will allow users to filter and drill down into specific cycle details.

### 1.3. Cost Tracking

- **Purpose:** Provides a breakdown of operational costs related to agent usage, resource consumption, and overall system metrics.
- **Key Elements:**
    - Overall System Metrics: Displays real-time CPU usage (fetched from Prometheus via a proxy API).
    - Cost Breakdown: A categorized list of costs by agent type, Qdrant usage, Loki storage, etc. (dummy data for now).
- **Interactions:** Metrics are refreshed periodically.

### 1.4. Queue Visualization

- **Purpose:** Shows the state of pending, active, and completed work items within the Director's queue.
- **Key Elements:**
    - Placeholder for queue visualization.
- **Interactions:** Future features will include filtering by repository or labels.

### 1.5. Agent Logs

- **Purpose:** Allows users to query and view logs generated by the Director and its agents, powered by Loki.
- **Key Elements:**
    - A text input for entering Loki queries.
    - A button to fetch logs based on the query.
    - A log viewer component displaying the fetched logs.
- **Interactions:** Users can input Loki-compatible queries to filter logs.

## 2. API Documentation

The dashboard interacts with the Swarm Director and other backend services through several API endpoints. For security and to manage CORS, these external API calls are proxied through Next.js API routes within the dashboard application.

### 2.1. WebSocket Endpoint

- **Purpose:** Real-time updates from the Director.
- **Endpoint:** `ws://[DIRECTOR_HOST]:8080/ws` (configured via `NEXT_PUBLIC_DIRECTOR_WS_URL`)
- **Proxy Status:** Directly connected from client-side via `useDirectorWS` hook.
- **Messages:** JSON objects with `type` and `payload` fields.

### 2.2. Prometheus Metrics API

- **Purpose:** Fetching system and application metrics.
- **External Endpoint:** `http://[PROMETHEUS_HOST]:9090/api/v1/query` (configured via `DIRECTOR_PROMETHEUS_URL`)
- **Next.js Proxy Endpoint:** `/api/prometheus`
- **Method:** `GET`
- **Query Parameters:**
    - `query` (string, required): The PromQL query to execute.
- **Response:** Proxies the JSON response from the Prometheus API.

### 2.3. Loki Logs API

- **Purpose:** Querying and filtering application logs.
- **External Endpoint:** `http://[LOKI_HOST]:3100/loki/api/v1/query_range` (configured via `DIRECTOR_LOKI_URL`)
- **Next.js Proxy Endpoint:** `/api/loki`
- **Method:** `GET`
- **Query Parameters:**
    - Supports standard Loki query parameters (e.g., `query`, `start`, `end`, `limit`, `direction`).
- **Response:** Proxies the JSON response from the Loki API.

### 2.4. Director Control API

- **Purpose:** Sending commands to control the Director's orchestration (e.g., trigger cycles, pause/resume).
- **External Endpoint:** `http://[DIRECTOR_HOST]:8080/control` (configured via `DIRECTOR_CONTROL_URL`)
- **Next.js Proxy Endpoint:** `/api/director/control`
- **Method:** `POST`
- **Request Body (JSON):**
    - `command` (string, required): The command to execute (e.g., "triggerCycle", "pauseOrchestration", "resumeOrchestration").
    - `payload` (optional): Any additional data required by the command.
- **Response:** Proxies the JSON response from the Director's control API.
